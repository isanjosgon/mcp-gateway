# Security Policy

## Supported Versions

Security fixes are applied to the latest released minor version.

## Reporting a Vulnerability

Please **do not** open public issues for security vulnerabilities.

Instead:
1. Use GitHub **Private vulnerability reporting** (recommended), if enabled in this repository.
2. Otherwise, contact the maintainers privately (add a contact method here).

Include as much detail as possible:
- Steps to reproduce
- Impact assessment (what an attacker can do)
- Affected versions / commit SHA
- Any proof of concept (PoC) (keep it minimal)

## Disclosure

We will acknowledge receipt, investigate, and coordinate a fix. We aim to publish a patch release as soon as feasible.
